# Verifica 1 C

## Cosa devi fare
1. Clona il repository.
2. Rinomina la cartella della verifica in `verifica-1-C-nome-cognome`, usando il tuo nome e cognome.
3. Completa la verifica lavorando solo nei file indicati dalla consegna.
4. Prima di consegnare elminina la cartella `node_modules`.
5. Alla fine crea un archivio ZIP della cartella, ottenendo il file: `verifica-1-C-nome-cognome.zip`.
   Lo ZIP deve contenere tutta la cartella della verifica con i file modificati.
6. Carica lo ZIP su Google Classroom.

## File che puoi modificare
- src/models/types.ts
- src/exercises/task1.ts
- src/exercises/task2.ts
- src/exercises/task3.ts
- src/debug/debug-01.ts
- src/debug/debug-02.ts
- theory/quiz.md

## File che non devi modificare
- README.md
- package.json
- tsconfig.json
- src/main.ts
- src/data/mockData.ts

## Regole tecniche
1. Non cambiare nomi di file e cartelle.
2. Non cambiare i nomi delle funzioni già presenti.
3. Non rimuovere import/export esistenti.
4. Completa i TODO senza riscrivere tutto da zero.
5. Nei file di debugging aggiungi commenti brevi con `// FIX:` vicino alle correzioni principali.
6. Nel file theory/quiz.md devi lasciare una sola risposta corretta per domanda, marcata con `[x]`.

## Avvio del progetto
Apri il terminale nella cartella della verifica e usa:

```bash
npm run build
npm run start
```
